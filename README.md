# Bézier & B-Spline Curve Rendering System (Taichi-based)

## 1. 项目概述

本项目基于 Python 与 Taichi 框架，实现了一个**交互式二维曲线建模与渲染系统**。系统围绕经典曲线模型——Bézier 曲线与 B 样条曲线展开，完整覆盖了从数学建模、数值计算到 GPU 光栅化渲染的全过程。

项目不仅实现了基础曲线绘制功能，还进一步引入：

* 基于 **De Casteljau 算法** 的 Bézier 曲线求解
* 基于 **Cox–de Boor 递归公式** 的全局 B 样条曲线
* 基于距离权重的 **抗锯齿（Anti-Aliasing）光栅化**
* CPU / GPU 分离的高性能渲染管线

该系统可用于直观分析不同曲线模型的几何特性，并验证其控制行为差异。

---

## 2. 数学基础

### 2.1 Bézier 曲线（De Casteljau）

Bézier 曲线通过递归线性插值构造：

[
P(t) = (1 - t) P_i + t P_{i+1}
]

通过逐层插值递归计算，最终得到曲线点：

[
P(t) = \sum_{i=0}^{n} P_i B_i(t)
]

其核心特性为：

* **全局控制性（Global Control）**
* 所有控制点共同影响整条曲线

---

### 2.2 B 样条曲线（Cox–de Boor）

B 样条曲线采用基函数线性组合形式：

[
P(t) = \sum_{i=0}^{n} P_i N_{i,p}(t)
]

其中基函数 ( N_{i,p}(t) ) 通过 Cox–de Boor 递归定义：

[
N_{i,p}(t) =
\frac{t - u_i}{u_{i+p} - u_i} N_{i,p-1}(t)
+
\frac{u_{i+p+1} - t}{u_{i+p+1} - u_{i+1}} N_{i+1,p-1}(t)
]

本项目采用：

* **均匀开区间节点向量（Clamped Knot Vector）**
* **三次 B 样条（p = 3）**
* **端点插值（Endpoint Interpolation）**

---

## 3. 核心实现

### 3.1 曲线计算（CPU）

* Bézier：递归 De Casteljau
* B 样条：Cox–de Boor 全局计算
* 统一参数域：( t \in [0,1] )

所有采样点在 CPU 端一次性计算完成，避免频繁 CPU-GPU 通信。

---

### 3.2 GPU 并行光栅化

通过 Taichi Kernel 实现：

* 批量接收曲线点（Batching）
* 并行映射至像素坐标
* 高效点亮 Frame Buffer

---

### 3.3 抗锯齿（Anti-Aliasing）

针对传统光栅化中整数截断导致的走样问题，采用：

* **3×3 像素邻域采样**
* **像素中心距离计算**
* **指数衰减权重模型**

[
w = e^{-d \cdot \alpha}
]

实现效果：

* 消除阶梯边缘
* 提升曲线连续性与视觉质量

## 4. 系统交互设计

| 操作    | 功能                   |
| ----- | -------------------- |
| 鼠标左键  | 添加控制点                |
| `c` 键 | 清空控制点                |
| `b` 键 | 切换 Bézier / B-spline |

系统支持动态交互，可实时观察曲线形态变化。

## 5. 实验现象与分析

### 5.1 Bézier 曲线

* 曲线受所有控制点共同影响
* 容易产生较大形变甚至回环
* 控制具有全局性

---

### 5.2 B 样条曲线

* 曲线更加平滑
* 不必经过所有控制点
* 修改单个控制点仅影响局部区域

👉 验证了：

> **B 样条曲线具备良好的局部控制特性**

---

### 5.3 抗锯齿效果

* 曲线边缘由离散像素点变为连续过渡
* 明显减少锯齿现象
* 提升视觉真实感
